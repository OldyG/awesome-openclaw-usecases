# 가족 캘린더 통합 및 가정 도우미

현대 가정은 업무, 개인, 공유 가족, 자녀 학교, 방과후 활동 등 다섯 개 이상의 캘린더를 여러 플랫폼과 형식으로 관리합니다. 단일 뷰가 없어 중요한 일정이 놓치기 쉽습니다. 동시에 장보기 목록, 식료품 저장고 재고, 약속 일정 조율 등 가정 관리가 흩어진 문자 메시지로 이루어져 묻히곤 합니다.

이 사용 사례는 OpenClaw을 상시 가동하는 가정 조정자로 바꿉니다: 캘린더를 아침 브리핑으로 통합하고, 메시지를 모니터링해 실행 가능한 항목을 감지하며, 공유 채팅 인터페이스를 통해 가정 물류를 관리합니다.

## 문제점

- **캘린더 분산**: 업무 캘린더는 공유가 제한되어 있습니다. 학교 캘린더는 PDF나 손으로 작성된 웹페이지로 옵니다. 캠프 일정은 이메일에 있습니다. 매일 아침 각각을 수동으로 확인하는 건 지속 불가능합니다 — "일정을 다른 캘린더로 복사하는 건 잘 작동하다가 내가 까먹으면 하나가 빠져버립니다." 
- **가정 조정 비용**: "우유 얼마 있어?"는 냉장고, 지하 저장고를 직접 확인한 뒤 다시 문자를 보내야 합니다. 이걸 일주일치 장보기로 곱하면 일이 커집니다.
- **놓친 약속**: 약속 확인 메시지가 문자로 오고 방치됩니다 — 캘린더 이벤트도 없고, 이동 시간 버퍼도 없고, 알림도 없습니다.

## 기능

- **아침 브리핑**: 모든 가족 캘린더를 통합해 선호하는 채널로 일일 요약을 전달합니다
- **메시지 상시 모니터링**: iMessage/문자 대화를 감시하고 약속으로 보이는 내용을 자동으로 캘린더 이벤트로 생성합니다(치과 확인, 미팅 일정 등)
- **이동 시간 버퍼 추가**: 감지된 약속 전후에 이동 시간을 블록으로 추가합니다
- **가정 재고 관리**: 냉장고/팬트리 항목의 실시간 재고를 유지하여 어느 파트너든 어디서나 조회할 수 있습니다
- **장보기 조정**: 레시피별 중복 재료 제거, 부족 항목 추적, 장보기 목록 생성
- **사진 입력**: 학교 캘린더나 냉동고 사진을 찍어 보내면 에이전트가 구조화된 데이터로 처리합니다

## 필요한 기술

- 캘린더 API 접근 (Google Calendar, Apple Calendar via `ical`)
- 메시지 모니터링을 위한 `imessage` 스킬 (macOS 전용)
- 공유 가족 채팅 인터페이스를 위한 Telegram 또는 Slack
- 재고 추적을 위한 파일 시스템 접근
- 물리적 캘린더 OCR을 위한 카메라/사진 처리

## 설정 방법

### 1. 캘린더 통합

OpenClaw이 모든 가족 캘린더 소스에서 데이터를 가져오도록 구성합니다:

```text
## Calendar Sources

아침 브리핑(08:00)에:

1. 내 Google Work Calendar 가져오기(read-only OAuth)
2. 공유된 Family Google Calendar 가져오기
3. 파트너의 캘린더 (공유 보기) 확인
4. ~/Documents/school-calendars/ 에 새 PDF가 있는지 확인 → OCR로 이벤트 추출
5. 최근 이메일에서 캘린더 첨부파일이나 초대 확인

하나의 브리핑으로 컴파일:
- 오늘의 이벤트(모든 캘린더, 출처별 색상 구분)
- 향후 3일간 일정(충돌 여부 확인)
- 어제 이후 추가된 새 이벤트
- 야외 이벤트를 위한 날씨 컨텍스트

Telegram/Slack 가족 채널로 전달합니다.
```

### 2. 상시 메시지 모니터링

이 부분이 핵심 차별점입니다 — 에이전트가 수동이 아닌 능동으로 감지하고 행동합니다:

```text
## Message Monitoring (HEARTBEAT.md)

15분마다:
1. 모든 대화의 새로운 iMessage 확인
2. 약속 패턴 감지:
   - "약속이 확정되었습니다..."
   - "[날짜] [시간]에 만날 수 있을까?"
   - "연습이 토요일 3시에 바뀌었어"
3. 감지 시:
   - 세부 정보로 캘린더 이벤트 생성
   - 전후로 30분 이동 버퍼 추가
   - 가족 Telegram에 확인 전송: "생성됨: 치과 약속, 화요일 14:00. 이동 시간 13:30-14:00 및 15:00-15:30 추가됨."
   - 파트너와 관련되면 초대 추가
4. 약속/약속 패턴 감지:
   - "금요일까지 보낼게"
   - "다음 주에 저녁 먹자"
   → 캘린더 홀드 또는 리마인더 생성
```

### 3. 가정 재고

```text
## Pantry Tracking

~/household/inventory.json 유지:
- 항목명, 수량, 위치(냉장고/팬트리/지하)
- 최종 업데이트 타임스탬프
- 부족 임계값

업데이트 방식:
- 사진: 사용자가 냉장고/팬트리 사진 전송 → 비전 모델이 항목 추출
- 텍스트: "달걀 다 떨어졌어" / "우유 2갤런 샀어"
- 영수증: 장보기 영수증 사진 → 재고 업데이트

조회: 어느 파트너든 Telegram으로 요청 가능:
- "버터 있어?" → 재고 확인 후 위치와 수량으로 응답
- "무엇이 부족해?" → 임계값 이하 항목 나열
- "장보기 목록 생성" → 부족 항목 + 레시피 필요 재료 컴파일
```

## 주요 통찰

- **능동적 감지 > 수동적 입력**: 가장 큰 가치 전환은 에이전트가 요청하지 않아도 행동하는 것입니다. 문자에서 약속을 감지하고 이동 버퍼를 추가해 캘린더 이벤트를 만든다는 점이 좋습니다.
- **Mac Mini가 최적**: 이 사용 사례는 홈 Mac Mini에서 실행하면 큰 이점이 있습니다 — iMessage 통합, Apple Calendar 접근, 항상 켜져 있는 상태
- **먼저 읽기 전용으로 시작**: 이벤트 생성이나 메시지 발송 같은 쓰기 작업을 활성화하기 전에 읽기와 모니터링부터 시작하세요
- **공유 Telegram 채널**: 양파트너가 에이전트의 동작을 볼 수 있어 신뢰를 쌓고 오류를 조기에 발견합니다
- **사진 입력은 과소평가됨**: 학교 캘린더나 냉동고 사진을 찍는 것이 입력보다 빠르고 비전 모델이 잘 처리합니다

## 영감 출처

이 사용 사례는 여러 커뮤니티 패턴을 결합합니다:

- **캘린더 통합**: HN 사용자 `angiolillo`가 [Hacker News 토론](https://news.ycombinator.com/item?id=46872465)에서 아침마다 업무, 개인, 가족, 자녀 학교 캘린더를 따로 확인하는 고통을 설명했습니다.
- **메시지 상시 모니터링**: [Sparkry AI](https://sparkryai.substack.com/p/24-hours-with-openclaw-the-ai-setup)가 문서화한 사례 — 아내가 치과 예약 문자를 받자 OpenClaw이 자동으로 30분 이동 버퍼를 추가한 캘린더 이벤트를 생성했습니다. 또한 [OpenClaw Showcase](https://openclaw.ai/showcase)에서 `@theaaron`이 채팅 기반 캘린더 관리를 "내가 사용해 본 LLM 활용 사례 중 최고"라고 언급했습니다.
- **가정 조정**: Brandon Wang의 [Clawdbot "Linguini"](https://brandon.wang/2026/clawdbot)는 홈 Mac Mini에서 실행되어 문자 후속 조치, 사진으로부터 캘린더 이벤트 생성, 냉동고 재고 처리, iMessage와 Slack을 통한 가정 조정 작업을 수행합니다.
- **팬트리 추적**: 여러 HN 사용자가 팬트리 재고 유지의 가치와 어려움을 논의했으며, `dns_snek`는 "내가 물건을 어디다 두었는지 5초만에 잊어버린다... 정말 큰 문제"라고 언급했습니다.
